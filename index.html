01 / How I Work

근거 위에서 만들고,
쉽게 전합니다

복잡한 정보를 누구나 이해할 수 있는 형태로 바꾸는 일을 좋아합니다. 어려운 예·적금 약관을 쉬운 말로 풀어주는 서비스를 만든 것도 그래서였습니다. 아래 세 가지는 프로젝트를 하며 몸으로 익힌 저의 작업 방식입니다.

/ 01

데이터로 검증하는 문제 해결

우대조건 텍스트를 정규식으로 뽑으려다 은행마다 다른 표현 앞에서 무너진 적이 있습니다. 그 뒤로는 로직을 짜기 전에 실제 데이터부터 열어봅니다. 추측으로 만든 코드는 데이터 앞에서 어긋나기 쉽다는 걸 몸으로 배웠기 때문입니다.

/ 02

실무에 맞게 적용하는 AI

상품 766건의 우대조건은 LLM 배치로 미리 구조화해, 추천 요청 때는 LLM 호출 없이 DB 쿼리만으로 바로 응답하게 만들었습니다. 실시간 챗봇은 답변 근거를 해당 상품 데이터로 한정해 지어낸 답을 막았습니다. AI는 쓰는 것 자체보다 어디에 어떤 형태로 쓸지 정하는 게 더 중요하다고 생각합니다.

/ 03

기획부터 배포까지

기획이 단단해야 ERD와 사용자 플로우가 명확해진다고 믿어서, 코드보다 기획을 먼저 다집니다. 그 위에서 DB 모델링과 API 설계, CI/CD 구축, 클라우드 배포와 도메인 연결까지 직접 해 봤기에, 지금 짜는 코드가 서비스 전체 어디에 놓이는지 이해하고 개발합니다.

02 / AI Workflow

AI를 다루는 방식

모두가 AI를 쓰는 시대에는, 어떻게 쓰는지가 결과의 차이를 만든다고 생각합니다. 저는 AI를 코드 생성기가 아니라 규칙 위에서 움직이는 협업 파트너로 씁니다. 자율주행 대회 2위도 이 네 단계를 반복해서 얻은 결과입니다.

01

합의

일을 시작하기 전에 목표·제약·규칙을 충분히 주고받으며 AI와 같은 그림을 맞춥니다.

02

규칙화

합의한 내용은 규칙 문서로 고정합니다. 세션이 바뀌어도 AI가 같은 기준으로 움직이게 하기 위해서입니다.

03

실행

규칙 위에서 작업을 진행합니다. AI는 근거 있는 제안을, 판단과 검증은 제가 합니다.

04

회고

발견한 문제와 수정 이력은 규칙에 다시 쌓습니다. 같은 실수를 두 번 하지 않기 위해서입니다.
