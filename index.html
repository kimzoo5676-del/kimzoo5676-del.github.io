01 / How I Work

근거 위에서 만들고,
쉽게 전합니다

복잡한 정보를 누구나 이해할 수 있는 형태로 바꾸는 일을 좋아합니다. 실제 데이터를 확인하고, 함께 일하는 사람들에게 배우며, 기획부터 배포까지 서비스 전 과정을 다룹니다.

/ 01

데이터로 검증하는 문제 해결

가정을 코드로 옮기기 전에 실제 데이터부터 확인합니다. 추측으로 만든 로직이 실제 데이터 앞에서 어긋나는 걸 여러 번 겪었기 때문에, 문제가 생기면 감이 아니라 데이터를 먼저 열어봅니다.

/ 02

묻고 배우는 협업

모르는 것을 아는 척하지 않습니다. 막히면 묻고, 더 잘하는 사람의 방식을 유심히 봅니다. 그렇게 배운 것은 다음 기회에 제 몫을 해내는 밑천이 됩니다. 협업은 일만 나누는 게 아니라 서로의 경험을 나누는 것이라고 생각합니다.

/ 03

기획부터 배포까지

기획이 단단해야 ERD와 사용자 플로우가 명확해진다고 믿어서, 코드보다 기획을 먼저 다집니다. 그 위에서 DB 모델링과 API 설계, CI/CD 구축, 클라우드 배포와 도메인 연결까지 직접 해 봤기에, 지금 짜는 코드가 서비스 전체 어디에 놓이는지 이해하고 개발합니다.

02 / AI Workflow

AI를 다루는 방식

모두가 AI를 쓰는 시대에는, 어떻게 쓰는지가 결과의 차이를 만든다고 생각합니다. 저는 AI를 코드 생성기가 아니라, 명확한 규칙 위에서 움직이는 협업 파트너로 씁니다.

01

합의

일을 시작하기 전에 목표·제약·규칙을 충분히 주고받으며 AI와 같은 그림을 맞춥니다.

02

규칙화

합의한 내용은 규칙 문서로 고정합니다. 세션이 바뀌어도 AI가 같은 기준으로 움직이게 하기 위해서입니다.

03

실행

규칙 위에서 작업을 진행합니다. AI는 근거 있는 제안을, 판단과 검증은 제가 합니다.

04

회고

발견한 문제와 수정 이력은 규칙에 다시 쌓습니다. 같은 실수를 두 번 하지 않기 위해서입니다.
