배경 & 문제

예·적금을 고를 때 은행마다 제각각인 우대조건과 약관은 사회초년생이 이해하기 어렵습니다. 복잡한 금리 조건을 대신 계산·정리해서, "내가 실제로 얼마를 받는지"를 직관적으로 보여주는 서비스를 만들고자 했습니다. 저는 백엔드와 AI를 맡아 데이터 수집부터 추천 로직, LLM 연동, 배포까지 담당했습니다.

한 일

  • Django REST Framework로 REST API와 JWT 인증을 구현하고, 회원·상품·가입·찜·커뮤니티 도메인의 DB(ERD)를 모델링했습니다.
  • 금융감독원(FSS) API에서 예·적금 상품 766건을 수집하고, 조회 프로필 기반 필터링 + 세후 실수령액 계산·정렬로 추천 알고리즘을 설계·구현했습니다.
  • LLM(GPT-5-mini)을 세 가지로 연동했습니다 — ① 은행별 자유 텍스트 우대조건을 표준 태그(JSON)로 배치 구조화, ② 약관을 쉬운 말로 요약, ③ 상품 상세 실시간 스트리밍 챗봇(근거 한정으로 할루시네이션 방지).
  • GitHub Actions로 CI/CD를 구성하고 AWS EC2(Nginx·Gunicorn)에 배포한 뒤 실제 도메인을 연결해, 로컬에서 프로덕션까지 서비스 생명주기 전체를 경험했습니다.

결과 & 배운 점

삼성청년SW·AI아카데미 1학기 관통 프로젝트 최우수상(서울2반 1등)을 받았습니다. 조건 입력 한 번으로 766개 상품을 세후 실수령액 기준으로 정렬해 추천하는 서비스를 배포했습니다(our-wish.site). 우대조건 구조화를 처음엔 정규식으로 시도했지만 은행별 표현 차이로 누락·오탐이 컸고, 실제 데이터를 직접 확인해 보편적인 우대조건만 표준 태그로 좁히는 방향으로 전환했습니다. 이 과정에서 가정하지 말고 데이터를 확인한다는 원칙과, LLM을 배치·실시간으로 나눠 응답 속도와 비용을 관리하는 방법을 배웠습니다.
